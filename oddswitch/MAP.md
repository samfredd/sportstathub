# OddSwitch Engine: Deep Project Map

This document provides an exhaustive, low-level breakdown of the OddSwitch Engine. It is the definitive guide for understanding the system's internals, data flow, and anti-detection mechanisms.

---

## 🗺️ Architectural Blueprint

OddSwitch is built as a **Distributed Async Service**. It separates the high-latency browser automation tasks from the high-throughput API layer.

### Core Stack
- **Web**: FastAPI (Async)
- **Task Orchestration**: Celery + Redis Broker
- **Browser Automation**: Playwright + System Chrome (Stealth)
- **Database**: PostgreSQL 16 + SQLAlchemy 2.0 (Asyncpg)
- **Caching**: Redis (Sliding window rate limiting + JSON storage)
- **Logging**: Structlog (JSON formatted for ELK/Datadog)

## 🔄 The 10-Step Pipeline Implementation

Every job processed by `app/workers/pipeline.py` flows through these exact class-level implementations:

| Step | Operation | Class / Method | Description |
| :--- | :--- | :--- | :--- |
| **1** | **Cache Check** | `RedisCache.get_translation` | Checks if the exact `source:code` has been mapped to `target` recently. |
| **2** | **Resolve** | `ResolverWorker.resolve` | Acquires a browser from `BrowserManager` and scrapes the source booking code. |
| **3** | **Normalize** | `NormalizerWorker.normalize` | Converts raw data to `CanonicalSlip`. Resolves team aliases and market names. |
| **4** | **Match Events** | `MatcherWorker.match_events` | Performs DB lookup on `event_mappings`. Falls back to `SequenceMatcher` fuzzy logic. |
| **5** | **Translate** | `TranslatorWorker.translate_markets` | Maps source market/selection IDs to target bookmaker equivalents. |
| **6** | **Score** | `ConfidenceScorer.score` | Calculates `(event_match * market_match)` confidence. Assigns `TranslationStatus`. |
| **7** | **Rebuild** | `RehydratorWorker.rebuild` | Constructs a JSON payload structured for the target bookmaker's internal API. |
| **8** | **Generate** | `RehydratorWorker.generate_code` | Calls target adapter's `generate_booking_code` to interact with bookmaker APIs. |
| **9** | **Store** | `JobRepository.complete` | Updates `translation_jobs` in Postgres with the final JSON result and `COMPLETED` status. |
| **10** | **Notify** | `NotifierWorker.notify` | Triggers the `callback_url` via async POST if provided in the initial request. |

## 🧠 Data Intelligence (Normalization & Matching)

### Canonical Hashing (`app/schemas/canonical.py`)
The system uses a **Content-Addressable** approach. A `CanonicalSlip` hash is generated by:
1.  Sorting all legs by event name/date.
2.  Removing all bookmaker-specific IDs and **Current Odds**.
3.  SHA-256 hashing the resulting JSON.
*This ensures that even if odds change, the core "intent" of the slip is recognized as a cache hit.*

### Team Alias Resolution (`app/db/repository.py`)
The `MappingRepository.find_canonical_name` uses a hierarchical lookup:
- **Level 1**: Bookmaker-specific alias (e.g., "Man Utd" -> "Manchester United" for SportyBet).
- **Level 2**: Universal alias (e.g., "Man Utd" -> "Manchester United" for all).
- **Level 3**: Identity fallback (returns original name).

### Confidence Thresholds (`app/workers/confidence.py`)
- `SEMANTICALLY_EQUIVALENT` (> 0.9): Safe for auto-execution.
- `APPROXIMATE` (0.7 - 0.9): Requires manual review or warning to user.
- `PARTIAL` (< 0.7): One or more legs failed to match reliably.

## 🕵️ Stealth & Anti-Detection (`app/browser/`)

The automation layer is designed to bypass **Cloudflare (Turnstile)** and **Akamai** bot detection.

### Browser Lifecycle (`manager.py`)
- **System Chrome**: Launched with `channel="chrome"` to ensure a real TLS JA3 fingerprint. Playwright's default Chromium is often blacklisted.
- **Context Recycling**: Each browser context is destroyed after 20 uses to prevent memory leaks and tracking via long-lived cookies.
- **Proxy Rotation**: Integrated proxy support with bad-proxy detection.

### Fingerprint Spoofing (`stealth.py`)
- **WebGL/Canvas**: Injects subtle "noise" into rendering buffers so that every browser instance has a unique, non-automated fingerprint.
- **Navigator Shield**: Masks `navigator.webdriver`, spoofs `languages`, `hardwareConcurrency`, and `deviceMemory`.
- **Human Behavior**: Uses Bezier curves for mouse movement and Gaussian-distributed typing delays.

### Bet9ja Specifics (`adapters/bet9ja.py`)
- **Print Interception**: Suppresses `window.print()` via `add_init_script` to prevent the browser from hanging.
- **Internal API Hijacking**: Instead of clicking UI elements, it uses `evaluate()` to call Bet9ja's internal `BookABetV2` and `PalimpsestAjax` APIs directly using the browser's authenticated session.

## 🗄️ Database Schema & Redis Patterns

### PostgreSQL (`app/db/models.py`)
- **`translation_jobs`**: `ULID` PK, `status` (queued, processing, completed, failed), `result_json` (JSONB), `callback_url`.
- **`event_mappings`**: Maps `source_event` + `source_bookmaker` to `target_event`. Includes `confidence` score.
- **`team_aliases`**: `alias` vs `canonical_name`.
- **`api_keys`**: Hashed keys with `rate_limit_override` capabilities.

### Redis Key Structure (`app/cache/redis_client.py`)
- `bc:{bookmaker}:{code}`: Scraped raw slip data (TTL 5m-1hr).
- `slip:{hash}`: Cached `CanonicalSlip` JSON.
- `tx:{hash}:{target}`: Cached final translation (the "Golden Result").
- `dedup:{src}:{code}:{tgt}`: Lock to prevent dual-processing of the same request.
- `ratelimit:{identity}:{window}`: Sliding window counters for API throttling.
